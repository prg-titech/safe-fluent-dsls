# Use Case: Check for EDSL Syntax Errors.

## Actor: Downstream Developer (Fluent API User)

## Summary: Embedded language chunks are parsed and any syntax errors returned.

## Preconditions: Up-To-Date Embedded language chunks were found.

## Postconditions: An AST of the parsed language chunks / found syntax errors are stored.

### Basic Flow:

1. The actor updates an embedded language chunk
2. The system parses the embedded language chunk and when successful,
stores the AST, and otherwise stores the syntax errors, with
error type, error location and error message.
3. The system displays all found errors to the actor. The display location
is the same as each error location. The display items are:
Error Severity ("Error", "Warning", etc.) and Error Message.

### Alternative Flow: None

### Notes: None

### Example 1: SQL in Java

A simplified SQL query adheres to the grammar

```bnf
Query ::= "SELECT" {Selection || ","}+
          "FROM" id
          Where? ";" ;
          
Selection ::= "*" | id ;
          
Where ::= "WHERE" Exp ;

Exp   ::= id | nat | Exp "==" Exp ;

token id ::= [a-zA-Z_][a-zA-Z0-9_]* ;

token nat ::= [1-9][0-9]* ;
```

This can be transpiled to the following Java fluent API

```bnf
Query ::= .select(String... selections)
          .from(String table)
          Where?
          .end() ;

Where ::= .where(String exp)
        | .where() Exp ;
        
Exp ::= .columnId(String id) | .nat(int i) | Exp .equals() Exp ;
```

The following three method chains adhere to the given grammar:

```java
Query query = SQL.select("*")
                 .from("Students")
                 .end();
```

```java
Query query = SQL.select("name", "birth_year")
                 .from("Students")
                 .where().columnId("birth_year").equals().nat(2005)
                 .end();
```

```java
Query query = SQL.select("name", "birth_year")
                 .from("Students")
                 .where("birth_year == 2005")
                 .end();
```

The following examples do not adhere to the given grammar:

```java
Query query = SQL.select("*")
                 .from("Students")
                 .from("Teachers")
                 .end();
```

In this case, the second call to from is invalid.

```java
Query query = SQL.select("1 + 5")
                 .from("Students")
                 .end();
```
